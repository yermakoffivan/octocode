## Answer

- `notFound()` is defined in `packages/next/src/client/components/not-found.ts` at line 23 (`export function notFound(): never {`).
- It throws an `HTTPAccessFallbackError` whose `.digest` is set to `${HTTP_ERROR_FALLBACK_ERROR_CODE};404`. The throw line is:
  ```
  throw error
  ```
  preceded by:
  ```
  ;(error as HTTPAccessFallbackError).digest = DIGEST
  ```
  where `DIGEST = \`${HTTP_ERROR_FALLBACK_ERROR_CODE};404\``.
- The thrown error is caught in `packages/next/src/client/components/http-access-fallback/error-boundary.tsx` in the static method `getDerivedStateFromError` of the `HTTPAccessFallbackErrorBoundary` class. That method calls `isHTTPAccessFallbackError(error)`, extracts the HTTP status via `getAccessFallbackHTTPStatus(error)`, and returns `{ triggeredStatus: httpStatus }`. When `triggeredStatus` equals `HTTPAccessErrorStatus.NOT_FOUND` (404) the `render()` method renders the passed `notFound` React node (the segment's `not-found.tsx`) instead of children.
