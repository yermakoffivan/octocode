## Answer

- `revalidateTag` is defined server-side in `packages/next/src/server/web/spec-extension/revalidate.ts` at line **36**:
  ```ts
  export function revalidateTag(tag: string, profile: string | CacheLifeConfig) {
  ```

- When called, it delegates to the internal `revalidate()` helper which writes to `store.pendingRevalidatedTags` (the `WorkStore`):
  ```ts
  store.pendingRevalidatedTags.push({ tag, profile })
  ```
  (`store` comes from `workAsyncStorage.getStore()`.)

- The server reads `pendingRevalidatedTags` in `packages/next/src/server/lib/incremental-cache/index.ts` in the **`get()`** method of the `IncrementalCache` class (line **451** and **514**):
  ```ts
  workStore?.pendingRevalidatedTags?.some((item) => item.tag === tag)
  ```
  This check runs on every cache lookup to decide whether a cache entry should be skipped (treated as stale/expired) because one of its tags was invalidated in the current request. If the tag matches, the entry is bypassed and fresh data is fetched.
