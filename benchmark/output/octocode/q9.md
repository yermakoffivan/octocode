## Answer

- **Repository**: The official Next.js AI agent evaluations live in the `evals/` subdirectory of `vercel/next.js` (not a separate repo). The README states:
  > "Agent evals for Next.js. Each eval is a small Next.js app + a prompt + assertions. We run the prompt through a coding agent in a sandbox and check what it wrote. The point: find places where agents get Next.js wrong because their training data is stale, then fix it by shipping better docs in the `next` package itself."

- **Files making up one eval case** (from `evals/README.md`):
  - `PROMPT.md` — the prompt given to the coding agent (describes a symptom/goal like a real user would)
  - `EVAL.ts` — vitest assertions run against files the agent wrote (regex the source)
  - `app/` (or `pages/`) — the starting state fixture the agent is given to edit

- **Eval IDs by category** (from listing `evals/evals/`):
  - **Caching**: `agent-029-use-cache-directive`, `agent-032-use-cache-directive`, `agent-037-updatetag-cache`, `agent-040-instant`
  - **Proxy/Middleware**: `agent-031-proxy-middleware`, `agent-039-indirect-proxy`
  - **Request APIs**: `agent-034-async-cookies`, `agent-035-connection-dynamic`
  - **Revalidation**: `agent-037-updatetag-cache`, `agent-038-refresh-settings`
