# Octocode benchmark suite (source)

Benchmark source folders are split by measurement surface: offline engine
checks, offline CLI metadata checks, optional comparisons, live Octocode tool
questions, and cross-repo probes. Engine checks exercise the shipped
`octocode-engine` napi binary; CLI checks exercise built agent-facing help, tool
schemes, and OQL schema output. This suite lives in the private
`@octocodeai/octocode-benchmark` workspace package.

**For the run commands, folder-by-folder index, and full check reference, see
[`../BENCHMARK.md`](../BENCHMARK.md)** — that file is the canonical agent guide
and the one place those tables are maintained; this page only covers what is
specific to this `benchmark/` source folder and not duplicated there.

The canonical, machine-verified full-format-support matrix (AST / signature /
LSP / minify, every extension) is generated by `yarn support:gen`
(`check-matrix.mjs --write`) into
[`docs/LSP_SERVER_LIFECYCLE.md` → Full format support matrix](https://github.com/bgauryy/octocode/blob/main/docs/LSP_SERVER_LIFECYCLE.md#full-format-support-matrix)
— that generated doc, not a table here, is what the repo root README points to.

## Real samples, not invented

The `ast/` and `lsp/` samples (in the shared [`samples/`](./samples/)
directory) are real third-party source files fetched from public repositories
(TypeScript compiler, Rust core, Spring, LLVM, nvm, Bootstrap, Home
Assistant, …). Each `manifest.json` (`ast/manifest.json`, `lsp/manifest.json`)
records the exact repo, branch, path, byte size, and sha256 so a sample can be
re-fetched or audited; `_engine.mjs`'s `loadManifestSamples`/`readSample`
enforce those hashes at check time so a corrupted/edited/truncated fixture
fails loudly instead of passing silently. Files retain their upstream licenses
and are used only to exercise the engine.

## Notes

- Tree-sitter is the **only** signature path; there is no regex/heuristic
  fallback (the former `src/signatures/heuristic.rs` was orphaned and deleted).
  Structural-only grammars (HTML/CSS/SCSS/Less/Scala/JSON/YAML/TOML) produce no
  signature outline by design.
- Scala has a grammar (structural search works) but **no configured language
  server**, so it appears in `ast/` but not `lsp/`.
- The LSP check verifies wiring + native (in-process) semantics only. Real
  server-backed features need the external servers installed; the check reports
  their availability but never fails on a missing server.
