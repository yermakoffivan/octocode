The engine is portable — not a vendor CLI wrapper.

What we did not pick

Vendor-first wrapper

  • Prompt and memory behavior hidden inside the tool
  • Context shape changes with the vendor
  • Tool schema becomes hard to preserve
  • Model swaps become integration work
What we picked

ADK + MCP

  • Agents, tools, and sessions stay explicit
  • Models can change without rewriting tool contracts
  • Octocode remains the evidence interface
  • Workflows stay testable in isolation
No vendor CLIs

We avoid wrapping claude, gemini, or openai CLIs as the product interface. The durable surface is the research contract: tools, context, evidence, and output stay stable.

Keep this short: the point is not framework tribalism. The point is portability and durable evidence contracts.
