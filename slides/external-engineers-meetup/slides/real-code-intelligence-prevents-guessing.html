When agents meet a real codebase

Six predictable ways the guessing machine breaks down.

01
Greps one repo
Misses the implementation in the sibling service that owns the data.
02
Stops at text matches
Finds the string, but misses the caller and the actual owner.
03
Skips the call graph
Proposes changes without seeing who depends on the code.
04
Ignores git history
Reverts the bug fix that was added three years ago.
05
Floods its own context
Pastes 2,000-line files into the prompt and forgets the goal.
06
Hallucinates the code
Invents APIs, types, and file paths that don't exist.
Land each failure as a specific gripe an engineer in the room has lived through. Bigger models don't fix any of these — tokenizers weren't built for multi-context flows.
