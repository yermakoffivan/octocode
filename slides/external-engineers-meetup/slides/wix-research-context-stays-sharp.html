Compaction built for research, not for chat.

Why custom

Generic compaction drops the things research depends on: file:line anchors, request IDs, failed searches, open gaps. Our compactor enforces a 7-heading structure that preserves them.

Generic chat compaction

Drops what research needs

  • Loses exact anchors: owner/repo:path:line, requestId, artifactId
  • Forgets failed searches — the same dead end gets retried
  • Overwrites prior summary instead of merging
  • Fits chat tone, not investigation state
Research-aware compaction

Resumable research state

  • Preserves anchors verbatim — file:line, URLs, FQDNs, package@version
  • Keeps no-result searches and open gaps explicitly
  • Re-compacts: merges previous summary with new history, drops superseded claims
  • Tasks persist across compactions via task_ops

Why we built it: generic chat-style compaction is wrong for research. Anchors (file:line, request IDs, artifact IDs, URLs) must survive verbatim; failed searches must be remembered so the same dead end is not retried; re-compaction must merge previous summary with new history rather than overwrite. The compactor enforces a strict 7-heading markdown structure (Research Progress → Searches Performed) every time. Triggers are size-based — 455K chars for the researcher (gateway-aware), 120K for helpers. Last 8 turns are always kept intact. Tasks persist across compactions through task_ops, so the researcher resumes with full state.
