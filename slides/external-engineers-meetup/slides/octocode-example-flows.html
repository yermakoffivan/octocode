Octocode in motion — two flows.

The agent stays in control of how much context it pulls. Octocode surfaces shape; the agent decides depth.

Search & paginate a file

Flow 01
  1. Search GitHub for a keyword Locate candidate files by symbol or phrase — not full clones.
  2. Fetch the file locally Octocode returns totalLines and a window — not the whole blob.
  3. Agent paginates as it likes Reads the next offset + limit only when the trail demands it.
Depth on demand — no upfront file dump.

Read a PR with intent

Flow 02
  1. List changed files Shape first — paths, sizes, change counts — before any diff.
  2. Agent picks what matters Skips lockfiles, generated code, untouched neighbours.
  3. Pull only the relevant hunks Paginated diffs keep the window honest — review, not bloat.
Intent-driven reads — no context flood.
Concrete payoff of the protocol: in flow 1 the agent searches GitHub, fetches one file, and pages through it on its own terms. In flow 2 the agent gets the shape of a PR first, then decides which files actually deserve attention. Same rule both times: Octocode hands back enough to choose, the agent chooses depth.
