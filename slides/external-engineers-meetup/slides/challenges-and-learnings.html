Four lessons that bent the system into shape.

Lesson 01
Library choice The foundation decides your ceiling.
  • Google ADK shipped only basic compaction and no parallel MCP tool calls.
  • We forked it and built our own compaction + parallelism layer.
Lesson 02
Context engineering Every tool you add costs the agent.
  • More tools = more prompt weight, slower convergence.
  • Good prompts are written by humans — not generated by AI.
Lesson 03
Human in the loop LLMs take the shortest path — not always the right one.
  • Quality validation is constant, not a launch checkbox.
  • Context management is the lever: schemas, helpers, compaction.
Lesson 04
The LLM knows Audit your prompts with the model that runs them.
  • Ask the model what it understood — gaps surface immediately.
  • Cheap iteration loop: write → ask → rewrite.
Net

None of this is one-shot work. The system improves as the library, prompts, tools, and evals all improve together.

Four lessons from the road. (1) Library choice: Google ADK shipped only basic compaction and could not fire parallel MCP tool calls — we forked it and built our own. The foundation decides your ceiling. (2) Context engineering: every tool you add costs the agent in prompt weight and decision pressure. Good prompts are not generated by AI; they are written, tested, and curated by humans. (3) Human in the loop: LLMs take the shortest path, which is not always the correct one. Quality validation is constant; evals and optimizations are ongoing; context management is the primary lever — schemas, helpers, compaction, smart prompts. (4) The LLM knows: ask the model what it understood from your prompts, use it to validate and polish content. Net: none of this is one-shot work; it improves as everything around it improves.
