Two bands. (1) Tools: the full MCP toolbelt the researcher calls directly for cheap, single-call decisions — code, runtime, logs, errors, data, docs, tickets, experiments, placements, memory, tasks. (2) Helpers: four isolated sub-agents, each scoped to a strict tool subset (or built per-call for adaptive); each card lists the trigger conditions. Helpers run in fresh sessions and return typed findings, not transcripts — that is what keeps the parent context lean.
