Bilbo at altitude — four stations, three typed handoffs, one reason for each. (0) The Query is callable by anything — a service, a web client, or another agent — same contract end-to-end. (1) Query anchors scope and intent — without it the agent wanders. (2) Planner disambiguates the ask and pre-fetches memory + Wix docs + Jira in parallel (PlannerOutputSchema in agent-contracts.ts) so the researcher never starts from guesses. (3) Researcher runs plan→call→observe across 11 MCPs and delegates depth to isolated helpers — depth without polluting parent context. (4) Answer is shaped to the contract (markdown / JSON / CSV / code) with file:line citations — reviewable, replayable, resumable. Next slides zoom into each stage.
