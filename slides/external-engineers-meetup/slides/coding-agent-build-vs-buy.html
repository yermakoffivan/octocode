Custom flow vs. CLI agent.

We could have run on top of an existing CLI agent — Claude Code, Codex, and friends. We chose a custom flow so the model stays a config choice, not a vendor lock-in.

Option A — CLI agent e.g. Claude Code / Codex
Run on top of a vendor CLI.

Plug into a ready-made coding agent and ship fast. The trade-off: the loop, prompts, and tool surface live inside someone else’s product — when the CLI changes its behavior, pricing, or model, our flow changes with it.

Option B — custom flow What we chose
Own the planner, researcher, and tools.

Planner, researcher, helpers, and tool contracts are ours. Models — Claude, GPT, Gemini — slot in behind a stable interface, so we can evaluate vendors side-by-side and swap without rewriting the system.

Pro Customizability
Con Maintainability
Why it pays off

The model is a config choice, not a dependency. When Claude Code, Codex, or any CLI shifts, our loop keeps running — we just point it at a different model.
