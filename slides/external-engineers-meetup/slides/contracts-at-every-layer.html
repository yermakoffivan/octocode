Common tool protocols matter — and lift agent quality.

A shared contract across every layer gives the agent a cognitive scaffold: fewer hallucinations, cleaner tokens, tighter control.

01 broadest
Agent prompts

Roles and workflow set before any tool is touched. Planner scopes, researcher follows evidence, helpers stay narrow.

02 per-mcp
MCP instructions

Each MCP carries its own playbook so the agent knows how its tools chain — discover, search, read.

03 per-tool
Tool schemas

Typed inputs, typed outputs, and “when to use” copy. The schema is the prompt.

04 per-call
Result hints

Every response carries a recommended next move, so the loop never stalls on “now what?”

Net

One protocol across all four layers means the agent reasons inside the contract instead of around it.

contracts > cleverness
Second lesson, and the one that paid the biggest quality dividend across the whole system. Common protocols at every layer give the agent a cognitive scaffold to reason inside. Four layers, broadest to most specific. (1) Agent prompts set the workflow at the loop level: planner scopes, researcher follows evidence, helpers stay narrow. (2) MCP instructions wrap each tool family with its own playbook so the agent knows how the tools chain. (3) Tool schemas are the entry-point guidance: typed inputs, typed outputs, and “when to use, when not to use” copy. The schema is the prompt. (4) Result hints close the loop: every response, even empty ones, carries a recommended next move. Net: hallucinations drop, tokens stay clean, control gets tighter. Contracts beat cleverness every time.
