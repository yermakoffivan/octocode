Simplified: title "Dynamic helpers" with the "Under research" badge. Three-step flow — researcher picks tools, helper built on the fly, one finding back. One concrete example: tracing a 500 error needs grafana AND code, so the researcher names both MCPs and we spin up a fresh helper scoped to exactly those. Two plain-language reasons: real questions need multiple tools at once, and pre-building every combination doesn't scale — so we build the helper just-in-time. Implementation note (for questions): ADK's LlmAgent has tools fixed at construction, so we build the agent inside the FunctionTool's execute callback. Honest status: per-call construction has real overhead, still measuring when it earns its cost vs. routing to a fixed helper.
