Context engineering is a daily engineering practice.

Prompts require human taste and touch!

01 · Prompts
Prompts matter Prompts are code — treated like code.

Versioned, reviewed, diffed. A prompt drift moves the system more than a model upgrade.

// rule

02 · Tool budget
More tools, more problems Every tool you add costs the agent.

More entries = heavier prompt weight, slower convergence, more pivot mistakes. Curate tool surface per agent.

// rule
scope tools to the role — planner, researcher, helper.
03 · Contracts
Schemas & protocols Typed payloads cross every agent boundary.

Planner→researcher, researcher→helper, helper→memory — findings, pitfalls, nextSteps. No free-text handoffs.

// rule

04 · Evals
KPIs, quality, evals Measured. Constantly. Or it drifts.

Cite-rate, pivot-rate, helper success, memory recall — tracked per release. Evals catch the drift a smoke test never will.

// rule
any new prompt or tool ships with an eval.
Lesson eight pulls together the day-to-day craft — what we now call context engineering. Four pillars. (1) Prompts matter. We treat prompts like code: versioned, reviewed, diffed. A prompt drift moves the system more than a model upgrade. The hard rule we settled on: good prompts are written by humans — not generated by AI. (2) Tool budget: every tool you add costs the agent. Heavier prompt weight, slower convergence, more pivot mistakes. We scope tool surface per role — planner, researcher, and helpers each see only what they need. (3) Schemas and protocols: typed payloads cross every agent boundary. Planner to researcher, researcher to helper, helper to memory — everything carries findings, pitfalls, next-steps. No free-text handoffs. If it crosses an agent boundary, it has a schema. (4) KPIs, quality, evals: measured constantly or it drifts. Cite-rate, pivot-rate, helper success, memory recall — tracked per release. Evals catch the drift a smoke test never will; every new prompt or tool ships with an eval. Open question we don't have an answer to: do agentic flows need a TDD-like discipline — eval-first, prompt second? It's the next thing we're trying. The craft is still forming.
