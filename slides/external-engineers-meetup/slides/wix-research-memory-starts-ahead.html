Memory matters — it's the foundation of organizational memory, quality, and learning. Three phases. (1) Save: at end of a clean run we emit one record — query, plan, findings, best-practices. Errored, too-short, and low-signal runs are skipped so the corpus stays clean. (2) Index: chunks are embedded and stored in Vespa — open-source vector DB, the same engine behind Yahoo Search; ANN retrieval makes recall fast regardless of corpus size, and partitioning keeps scopes separate. (3) Recall: the planner queries the KB with the new question and surfaces top-3 chunks above 0.70 relevance as `memoryContext`. The hard rule: memory is hints, not evidence — the org changes daily (services rename, contracts shift, owners move), so every recalled identifier or claim must be re-grounded in current code, runtime, or docs before it shapes the answer. Two more things matter at this layer. Self-review: because we save plans and best-practices alongside findings, the agent can re-read what previously worked and refine its own instructions and flows — it's not just remembering facts, it's remembering how to work. Continuous improvement: we follow results from the research flow as the measurement signal — which hints helped, which plans converged, which tool sequences wasted tokens — and that feedback tunes prompts, tools, and protocols. The agent gets sharper every run.
