Context Driven Development — the working name for how we extend the agent beyond code. The organization carries meaning that no repo or dashboard captures alone: which team owns this, what the vocabulary actually means, which DB tables and runbooks close which questions. Teams already maintain skills that encode their domain — the agent reads those skills + live data so it investigates with the same context a human teammate would.
