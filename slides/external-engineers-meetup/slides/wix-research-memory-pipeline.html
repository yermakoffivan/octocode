Previous slide was the idea — Memory Matters. This one is the flow. Two seams, one loop. Write path (researcher saves): at the end of every clean run the researcher bundles the query, the typed PlannerOutput, the ResearchOutput, AND a self-review — what worked, what to do differently next time. The bundle goes through an LLM that rewrites it into smart-guide chunks (references, code locations, anchors, gotchas) phrased for the next planner. Those chunks land in wix-chunk-service, which embeds them and writes to Vespa with partition tags so recall stays scoped. Read path (planner gets it as initial context): when a new run starts — example, "Where is the code in Wix that relates to Premium account?" — the planner queries the brain before its first thought. Top hits come back on PlannerOutput as `memoryContext`. Past plans, answers, and self-reviews feed straight into the new plan as initial context — anchors to verify, not facts to copy. Tie back to the previous slide: that's exactly why memory is hints — the org changes daily, so even the highest-relevance hit goes through re-grounding before it shapes the answer. The loop closes here: research saves → planner gets → researcher saves again → the system gets sharper every run.
