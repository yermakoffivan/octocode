Trade-offs are a context problem — not a model problem.
Some decisions still need a human — not because the model can't write words, but because the words have organizational consequences the model can't see. Two real failure modes drive this. (1) Bugs often have more than one valid fix — the agent will happily pick one, but picking the right one means knowing the legacy, the owners, the deadlines. (2) What looks like a bug is sometimes a product decision or a missing feature — the triage call isn't a code question. And left unbounded, the agent will try to solve anything: I've watched it propose altering DB tables to handle an edge case it should have flagged instead. So we draw the line: the agent investigates and reports cited evidence; the human frames the problem and picks the path. Trade-offs are a context problem, not a model problem.
