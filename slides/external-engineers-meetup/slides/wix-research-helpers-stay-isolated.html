Isolated helpers protect the parent context.

PurposeAbsorb deep-read cost — keep the researcher’s context sharp.

Each sub-agent runs isolated, absorbs the heavy reads in its own MCP scope, and reports back through a typed finding. The parent's context grows by one verdict — not a transcript.

Reason 01 Clean authority

Each helper sees only its own tools — cannot recurse, cannot widen scope.

Reason 02 Context budget protected

Tool calls, file reads, retries stay in the helper. Parent grows by one finding.

Reason 03 Decision-grade artifacts

Researcher ingests verified / likely / unverified + cited evidence + named gaps.

The point is not "helpers do work the researcher cannot do" — it is "helpers absorb the heavy reads so the researcher's context stays sharp." Each helper is locked to a single MCP scope: research_helper → octocode, trino_helper → trino, grafana_helper → grafana, adaptive_helper → caller-specified tools[] at call time. That isolation gives three outcomes: clean authority (no recursion, no scope widening), protected context budget (parent grows by one typed finding), and decision-grade artifacts (verified/likely/unverified + cited evidence + named gaps).
