The Researcher builds answers exclusively from evidence.

Executes the planner’s contract: runs plan → call → observe across the MCP toolbelt, delegates long chains to isolated helpers, and emits a grounded, cited envelope shaped to the outputType the planner asked for — not a transcript, not a guess.

  • contract-shaped · artifact matches outputType + outputTypeDetail exactly — no creative reshaping
  • depth via isolation · long chains run in helpers and return typed findings — parent context stays lean
Tools · Toolbelt 11 MCPs · 4 helpers
  • Direct MCPs · octocode, service-topology, api-docs, grafana, error-tracking, trino, jira, users-data, personalization, memory
  • Helpers · research_helper, trino_helper, grafana_helper, adaptive_helper — fresh session, scoped tools
  • Tasks · task_ops — long-running async work, resumable
Flow · One loop Plan → call → observe → emit
  • Plan from PlannerOutput: pick startingLane, target criticalUnknowns
  • Call the cheapest tool that can prove or disprove the current hypothesis
  • Observe: read response hints, branch on evidence, cite file:line
  • Delegate depth to helpers; resume on typed findings, not transcripts
  • Emit one envelope shaped to outputTypeDetail with citations

The Researcher is the execution agent — it consumes the PlannerOutput verbatim and runs the plan→call→observe loop until the envelope is filled. Three invariants frame it. Evidence-only: every value in the final artifact must trace back to a tool result — not model memory, not paraphrase. Contract-shaped: outputType and outputTypeDetail from the planner are honored exactly — markdown stays markdown, JSON validates, code compiles. Depth via isolation: when a thread of work is going to require many turns, it goes to a helper in a fresh session and returns typed findings, so the parent context never fills up with raw tool transcripts. The toolbelt is 11 direct MCPs plus four scoped helpers and task_ops for async work. The envelope itself is a strict zod schema — the artifact shaped to the contract, a one-paragraph summary, citations with file:line, typed findings with confidence, plus optional fields for open questions, helpers used, tasks spawned, and a needs_clarification status when the question cannot be answered without more input.
